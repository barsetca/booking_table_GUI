# Инструкция по размещению проекта на GitHub

## Шаги для создания репозитория

1. **Создайте новый репозиторий на GitHub:**
   - Перейдите на https://github.com/new
   - Введите название репозитория (например: `booking_table_GUI`)
   - Выберите публичный или приватный репозиторий
   - НЕ инициализируйте с README, .gitignore или лицензией (мы уже создали их)

2. **Инициализируйте git в локальной директории:**
   ```bash
   cd /home/chernyak/AI_projects/booking_table_GUI
   git init
   ```

3. **Добавьте все файлы:**
   ```bash
   git add .
   ```

4. **Сделайте первый коммит:**
   ```bash
   git commit -m "Initial commit: Система бронирования столов в ресторане"
   ```

5. **Добавьте remote репозиторий:**
   ```bash
   git remote add origin https://github.com/YOUR_USERNAME/booking_table_GUI.git
   ```
   (Замените YOUR_USERNAME на ваш GitHub username)

6. **Отправьте код на GitHub:**
   ```bash
   git branch -M main
   git push -u origin main
   ```

## Проверка перед отправкой

Убедитесь, что следующие файлы НЕ попали в репозиторий:
- `.env` (файл с паролями)
- `venv/` (виртуальное окружение)
- `__pycache__/` (кэш Python)
- `*.pyc` (скомпилированные файлы)

Все эти файлы должны быть в `.gitignore` и не должны быть закоммичены.

## После размещения

1. Обновите README.md, если нужно добавить ссылку на репозиторий
2. Добавьте описание проекта на странице репозитория
3. Добавьте теги (tags) для лучшей навигации
4. Рассмотрите возможность добавления скриншотов интерфейса

## Полезные команды

```bash
# Проверить статус
git status

# Посмотреть, что будет добавлено
git status --short

# Проверить .gitignore
git check-ignore -v venv/
```
